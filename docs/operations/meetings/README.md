# Meetings

Canonical home for all company, team, and 1:1 meetings.

## Structure

```
docs/operations/meetings/
  YYYY/
    YYYY-MM-DD-<type>-<participants>-<topic>/
      transcript.md
      outcomes.md
```

- `<type>`: company|team|1on1|ad-hoc
- `<participants>`: dash-separated first names in alphabetical order (e.g., ben-eric)
- `<topic>`: short slug (e.g., facebook)

## Templates

- Transcript: `docs/operations/templates/meeting/TRANSCRIPT_TEMPLATE.md`
- Outcomes: `docs/operations/templates/meeting/OUTCOMES_TEMPLATE.md`

## Conventions

- Use front matter with `date`, `title`, `type`, `participants`, `context`, and `tags`.
- Keep transcript verbatim but readable; outcomes should be organized into decisions, actions, projects, principles, and PRDs.
