# Liftoff — Onboarding in Four Hours

Agenda (4 hours)
1) Rules (30m): review `01-rules-of-autonomy.md`; sign‑to‑work.
2) Purpose & KPIs (30m): `00-purpose-and-scope.md`, `04-kpis-as-experiments.md`.
3) DSM & API (60m): data layers, pg_cron, freshness SLAs; API v1 + OpenAPI; MCP tools.
4) CI/CD & Observability (60m): canaries, rollback drills, error budgets; dashboards.
5) Hands‑on (60m): ship a small flagged change; link change notes and dashboards; verify SLOs.

Access checklist
- Repo access; CI/CD; environment secrets (read‑only); dashboards
- API keys for read‑only role; OpenAPI docs
- MCP tools; staging environment access

Acceptance
- Complete sign‑to‑work
- Ship a small change safely and attach evidence
- Demonstrate how to diagnose a freshness breach and rollback
